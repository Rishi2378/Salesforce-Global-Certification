# Day 9 — Salesforce Certification Achievements

Welcome to the Day-9 folder for the Salesforce Global Certification study series. This directory collects notes, study outcomes, evidence of progress (screenshots, certificates), and short reflections tied to Day 9 of the learning roadmap.

## What’s in this folder

- `notes.md` — concise study notes, tips, and key takeaways from Day 9 topics.
- `certificates/` — scanned certificates or exported PDFs that verify completed modules or badges.
- `screenshots/` — screenshots of practice tests, setup confirmations, or progress indicators.
- `resources/` — optional: links or PDFs for reference materials used on Day 9.

(If any of the above files or folders are missing, add them using the naming guidance below.)

## Purpose

This README explains the intended contents of the Day-9 folder and provides guidance for organizing artifacts so they are easy to review later or share with others.

- Keep study notes short and focused (one idea or command per bullet).
- Use descriptive filenames that include the date and topic (see examples below).
- Add a short one-line summary at the top of each note or image filename metadata (if supported by your tools).

## Naming conventions (recommended)

- notes.md — Day 9 study notes
- certificates/
  - `YYYY-MM-DD_certificate_<provider>_<topic>.pdf` (e.g., `2026-07-15_certificate_Salesforce_Admin.pdf`)
- screenshots/
  - `YYYY-MM-DD_screenshot_<short-description>.png` (e.g., `2026-07-15_screenshot_trailhead-badge.png`)
- resources/
  - `author - title - YYYY.pdf` or include a link in `notes.md`

## Example note template (for notes.md)

- Date: 2026-07-15
- Topic: Approval Processes / Flows / Automation
- Key takeaways:
  - Bullet 1: short statement
  - Bullet 2: command, formula, or UI path
- Action items:
  - Try practice challenge X
  - Create a small flow that does Y

## How to contribute

If you want to add a file to this folder:
1. Follow the naming conventions above.
2. Add a short description at the top of the file explaining what it is and why it was added.
3. Commit with a clear message like "Add Day-9 notes: approval processes" or "Add certificate: Salesforce Admin - 2026-07-15".

## Privacy

Be mindful when adding screenshots or PDF certificates that contain personal information. Remove or redact any sensitive information before committing, or add those files only to a private repository.
